# Story 1.5: 기존 자산 재사용 경계와 금지 행위 명문화

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 개발자,
I want **어떤 기존 코드가 "보존·wrapping 재사용" 대상인지**와 **절대 하면 안 되는 금지 행위 7가지**를 한 문서(`docs/qa/`)로 못박고, 그 문서가 secret 없이·필수 섹션을 갖춰 보존됐는지 자동 회귀 가드 테스트로 고정하고,
so that 이후 **모든 에픽(2~5)의 구현 에이전트**가 기존 배민/쿠팡 동작 보존 경계를 흔들지 않고, `project-context.md`의 56개 규칙을 최우선 권위로 삼아 일관되게 작업하며, 경계를 바꿔야 할 때는 임의 변경이 아니라 **ADR/예외 기록 절차**를 거치게 된다.

> **이 스토리의 성격(중요 — 1.1/1.2/1.4와 같은 "문서·산출물 스토리", 1.3 같은 코드 스토리 아님).** Epic 1(기준선 안전망, P0)의 **마지막 스토리**이며 spec **P0** 가드레일과 **FR-2**(기존 자산 재사용 보장)·**ADD-15/Forbidden Behaviors**(금지 행위)를 한 권위 문서로 고정한다. 산출물은 **① 재사용 경계·금지 행위 거버넌스 문서(`docs/qa/`), ② 그 문서를 지키는 회귀 가드 테스트(`tests/`)** — **신규 2파일뿐**이다. **제품 코드(`src/rider_crawl/`)는 한 줄도 바꾸지 않는다.** 이 스토리는 기존 코드의 경계를 **"문서화"** 할 뿐, 코드를 옮기거나 리네임하거나 wrapping을 **구현하지 않는다**(실제 wrapping/분리는 Epic 2~4 책임).
>
> **기준선 회귀 0.** 현재 HEAD(`478d760`) 기준 전체 스위트는 **538 collected/passed**(실측 확인 — 복사 금지, 본인이 `.venv/Scripts/python.exe -m pytest -q`로 재측정)다. 본 스토리는 신규 가드 테스트 케이스 수만큼만 passed가 증가하는 게 정상이고, 기존 통과 테스트가 새로 깨지면 실패다(NFR-20).
>
> **secret/식별자 비노출(NFR-5, ADD-15).** 커밋되는 거버넌스 문서·테스트에 실제 토큰/비밀번호/OTP/`chat_id`/전화/이메일/마스킹 안 된 운영 식별자 원문을 넣지 않는다. 모두 placeholder/가짜값만 쓴다.

## Acceptance Criteria

**AC1 — "보존·wrapping 재사용" 경계 명문화 (FR-2, implementation-contract Reuse And Replace)**
1. **Given** 보존해야 할 기존 자산이 있을 때 **When** 재사용 경계를 `docs/qa/`의 거버넌스 문서에 표(Keep / Required change)로 문서화하면 **Then** 다음 7개 자산이 **"보존·wrapping 재사용 대상"** 으로, 각각의 **현재 코드 위치**와 **허용되는 변경(wrapping/추가)·금지되는 변경**과 함께 명시된다 — **① 배민 parser/crawler**(`src/rider_crawl/parser.py`, `crawler.py` — 레거시 import/테스트 호환 유지), **② 쿠팡 parser**(`src/rider_crawl/platforms/coupang/`), **③ message renderer**(`src/rider_crawl/message.py` `render_current_screen_message`), **④ Telegram/Kakao sender**(`src/rider_crawl/sender.py` + `rider_crawl.messengers` registry), **⑤ 쿠팡 Gmail 2FA 로직**, **⑥ `run_once` 실행 경계**(`src/rider_crawl/app.py` `run_once`), **⑦ platforms/messengers registry**(`rider_crawl.platforms`, `rider_crawl.messengers`). [Source: implementation-contract.md Reuse And Replace(5-12), project-context §42-45, epics.md FR-2(24)]
2. **And** 기존 **공개 동작 4종 — (a) 렌더링 결과**(`test_message.py`/`test_coupang_message.py` 골든 테스트가 잠금), **(b) 저장 JSON 호환**(`ui_settings.json` `ensure_ascii=False, indent=2`·legacy 카카오 설정), **(c) 탭 9개 로딩**(`UiSettingsStore.load_all(max_tabs=9)`), **(d) 쿠팡 플랫폼 추론**(`test_architecture.py`/registry) — 을 **의도 없이 바꾸는 변경은 "실패(regression)"로 취급**한다고 문서에 못박는다. [Source: epics.md AC(342), project-context §47·59·68, FR-2(24)]
3. **And** 문서는 각 보존 자산에 **"required change"(예: 배민 parser는 정규화 Snapshot으로 wrapping + fixture 테스트 유지, Telegram sender는 중앙 webhook 전환·Agent별 getUpdates polling 제거)** 를 함께 적되, **그 변경의 실제 구현은 해당 에픽(2~4) 책임**임을 명시한다(본 스토리는 경계 문서화만, wrapping 구현 아님). [Source: implementation-contract.md Reuse And Replace(7-20), FR-Coverage-Map(155)]

**AC2 — 금지 행위 7가지 명문화 (ADD-15, operations-security-test-contract Forbidden Behaviors)**
4. **Given** 금지 행위를 명시해야 할 때 **When** 금지 목록을 문서화하면 **Then** 다음 **7개 금지 행위**가 각각 **금지 사유(코드/운영 근거)** 와 함께 빠짐없이 나열된다 — **① 탭 9→100 확장으로 스케일링**, **② 배민 휴대폰 인증 자동화/우회**, **③ 같은 Windows session에서 Kakao 2건 병렬 전송**, **④ 고객 간 Gmail token 공유**, **⑤ secret(token/password/OTP)을 로그·DB text 필드·스크린샷·config 파일·에러 메시지에 저장**, **⑥ 클라우드가 로컬 Chrome CDP 포트에 직접 접속**, **⑦ backoff/circuit breaker 없이 parser 실패를 빠르게 재시도**. [Source: operations-security-test-contract.md Forbidden Behaviors(87-94), epics.md ADD-15(143)]
5. **And** 각 금지 행위에 **올바른 대안**을 한 줄로 연결한다(예: ⑤→Story 1.3 `redaction.redact()` 재사용·`*_ref`만 저장, ③→KakaoSendJob FIFO queue + sender lock, ⑦→jitter·exponential backoff·circuit breaker). 대안의 **구현 책임 에픽**(Epic 2 secret_ref, Epic 4 Kakao queue, Epic 5 circuit breaker 등)을 괄호로 표기한다. [Source: implementation-contract.md Replace(16-20), epics.md FR-Coverage-Map(159·168·186), project-context §49·89·93]

**AC3 — 권위 기준·예외 기록 절차 명문화 (NFR-5, governance)**
6. **Given** 이 문서가 권위 기준으로 쓰여야 할 때 **When** 구현 에이전트 가이드를 작성하면 **Then** 문서는 **권위 계층**을 명시한다 — **최우선: `_bmad-output/project-context.md`의 56개 규칙**(에이전트가 코드 구현 전 먼저 읽고, 일반 관례보다 우선), 그 다음 본 거버넌스 문서·`architecture.md`·spec 계약. 확신이 없으면 **더 제한적인 선택**을 한다는 원칙을 인용한다. [Source: project-context.md(101-106), epics.md AC(350)]
7. **And** 경계/금지 규칙을 바꿔야 할 정당한 사유가 생기면 **임의 변경 금지**, 대신 **`architecture.md`의 ADR 또는 `project-context.md`에 예외를 기록**하고 근거를 남기도록 지시한다. 위반(보존 동작을 의도 없이 깸·금지 행위 도입)을 발견하면 그 작업을 **실패로 보고**하라고 명문화한다. [Source: epics.md AC(351), project-context.md(106)]

**AC4 — 거버넌스 문서 회귀 가드 테스트 (NFR-20, NFR-5 — Epic 1 산출물 가드 패턴)**
8. **Given** 이 스토리가 거버넌스 문서를 자동 회귀로 고정해야 할 때 **When** `tests/`에 가드 테스트(`tests/test_reuse_boundaries_doc.py`)를 추가하면 **Then** 거버넌스 문서가 **존재**하고, **AC1의 7개 보존 자산 식별 문자열·4개 공개 동작·AC2의 7개 금지 행위·AC3의 권위 계층(project-context 참조)·예외 기록(ADR/architecture) 섹션**이 모두 포함됐는지 단언한다. (`test_baseline_artifacts.py`·`test_manual_regression_runbook.py`와 동일한 **순수 파일 읽기** 테스트 — 외부 브라우저/Telegram/Kakao/Gmail/네트워크 미호출.) [Source: tests/test_baseline_artifacts.py, tests/test_manual_regression_runbook.py]
9. **And** 그 테스트는 문서(및 함께 커밋되는 텍스트)에 **실제 텔레그램 봇 토큰 형태(`\b\d{6,}:[A-Za-z0-9_-]{30,}\b`)·full email(`example.*` 가짜 도메인 제외)·한국 휴대폰 형태(전부 0인 placeholder 제외)·`chat_id=<digits>` 평문이 없음**을 정규식으로 단언한다(`test_manual_regression_runbook.py`의 `TELEGRAM_BOT_TOKEN_RE`/`EMAIL_RE`/`KOREAN_PHONE_RE`/`CHAT_ID_PLAINTEXT_RE` 재사용 권장). **주의: 파일 전체에 `redact()`를 통과시켜 "변화 없음"을 단언하지 말 것** — 의도된 placeholder/가짜 예시는 `redact`가 정상 마스킹하므로 거짓 실패를 낸다. 검사는 **실제 secret 패턴의 부재**로만 한다. [Source: tests/test_manual_regression_runbook.py(40-57), 1-4 AC4 #11]
