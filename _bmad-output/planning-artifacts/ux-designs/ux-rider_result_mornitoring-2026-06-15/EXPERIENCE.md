# rider_server 운영 대시보드 — Experience Spine (EXPERIENCE.md)

> 동작·정보구조·상태·인터랙션·접근성·여정의 정본. 시각 정체성은 `DESIGN.md`. 어떤 목업과 충돌해도 두 스파인이 이긴다.
> 시각 참조 목업: [`mockups/admin-redesign.html`](./mockups/admin-redesign.html) — 모니터링+관리 양 모드 클릭 가능 프로토타입.

## Foundation

- **폼팩터:** 반응형 웹. **데스크톱 : 모바일 ≈ 5:5**, 둘 다 1급. 벽걸이/대형 모니터 전용 모드는 없음.
- **UI 시스템:** 자체 완결형 — **darkmatter 토큰(인라인 CSS 변수) + Jinja2 + HTMX 2(CDN, 빌드 0)** 서버 렌더. 외부 컴포넌트 라이브러리 없음. `DESIGN.md` 가 시각 정본.
- **테넌시:** **단일 운영자/단일 고객.** 100여 개 업체(모니터링 대상)가 한 tenant 밑에 달림. tenant 전환 UI 불필요(`?tenant=` 는 내부 seam 으로만 잔존). 화면은 "업체 100개를 잘 보여주기"에 100% 집중.
- **쓰기 안전:** 대시보드 읽기는 read-only(`DashboardService`), 상태 변경은 별도 action/CRUD 라우터(OPERATOR↑ 권한 + actor·시각·diff audit). UI 재설계가 이 경계를 바꾸지 않는다.

## Information Architecture

상단 고정 바(앱바 + **모드 탭**) 아래 두 모드. 인증필요·수동액션 패널 같은 옛 별도 섹션은 **대상 워크벤치로 흡수**됐다.

| 서피스 | 진입 | 목적 |
|---|---|---|
| **모니터링** (기본) | 앱 열기 / 모드탭 | J1 트리아지. 요약(KPI 필터) + 대상 워크벤치 + (접힘) Agent·채널 |
| **관리** | 모드탭 | J2 등록/편집. 검색형 업체 목록 + `+ 새 업체 추가` 가이드 폼 + (접힘) 계정·채널·템플릿 |
| **업체 상세 드로어** | 어느 목록이든 행 클릭 / 텔레그램 알림 딥링크 | 한 업체의 왜·사실·이력·조치. 탭 `상세`(행 data로 즉시) / `편집`(열 때 lazy fetch). **업체 편집의 정본 경로** — 관리 모드 인라인 edit-row 대체. 모니터링·관리 공용 |
| **새 업체 추가** | 관리 ▸ `+ 새 업체 추가` | 가이드 폼 2-섹션(① 업체 만들기 → ② 채널 연결, 전부 드롭다운) |

- **요약 KPI = 필터.** 정상/주의/위험/중지 칩이 곧 대상 목록 필터. 숫자→항목 단절 해소.
- **딥링크 대상 상세.** `/admin/t/{target}` 류 안정 URL로 텔레그램 알림에서 바로 진입(모바일 1급). → 합성 참조: [`mockups/admin-redesign.html`](./mockups/admin-redesign.html). 스파인이 충돌 시 이김.

## Voice and Tone

마이크로카피. 브랜드 보이스/미감은 `DESIGN.md.Brand & Style`.

| Do | Don't |
|---|---|
| "지금 조치할 8곳" | "조치 필요 항목: 8" |
| "로그인 만료 · 인증 확인 필요" | "AUTH_REQUIRED" / "MissingPerformanceDataError" |
| "🟢 100곳 모두 정상 · 지금 조치할 항목 없음" | (빈 표) |
| "정말 '동대문엽기떡볶이 본점'을 비활성화할까요?" | "비활성화하시겠습니까?" |
| "갱신 실패 · 마지막 14:30 · 재시도 중" | (조용히 멈춘 화면) |
| 운영자 언어: "지금 수집" / "미리보기" | 개발자 용어: "test-crawl" / "dry-run" |

원칙: 사람 문장 우선, 기계 코드는 숨기되 필요 시 괄호 보조. 비난조 금지·행동 유도. **secret/토큰/OTP 는 어떤 화면·메시지·이력에도 표시하지 않는다(핸들만).**

## Component Patterns

행동 명세. 시각은 `DESIGN.md.Components`.

| 컴포넌트 | 사용처 | 행동 규칙 |
|---|---|---|
| **target-row** | 모니터링·관리 목록 | 행 전체 클릭 → 상세 드로어. 상황맞춤 1차 액션 버튼은 `event.stopPropagation`(행 클릭과 분리). 심각도 내림차순 자동 정렬(위험·중지 위로). **데스크톱 열은 고정폭 트랙(플랫폼·심각도 배지·액션 열)을 공유**해 행마다 정렬이 맞는다 — 글자수에 따라 어긋나면 안 된다(배지/액션 열에 `auto`/content-기반 폭 금지, 텍스트 열만 `minmax(0,fr)`로 가변+말줄임). |
| **신선도 줄** | target-row 사유 열 안 | 사유 아래 한 줄: `수집 {상대시간} · 전송 {상대시간} · 주기 {N}분`. 수집/전송 시각 = 신선도색(정상=무채/주의=warn/위험=crit) + 텍스트(색 단독 금지). 수집 기록 없으면 "수집 기록 없음", 전송 없으면 전송 절편 생략. **주기는 무채(muted)·작게** — 맥락 정보라 의미색 금지, 한눈 스캔(심각도·사유 1순위) 방해 금지. 사유 열(가변 텍스트)에 들어가므로 고정폭 트랙·정렬을 건드리지 않는다. |
| **상황맞춤 액션** | target-row · 드로어 | 상태→액션 매핑: 로그인만료/AUTH→**인증 확인** · 쿠팡 센터명 미설정→**센터명 설정** · 연속 수집실패→**인증 확인**+**재시도** · 중지→**활성화** · 수집지연→**지금 수집** · 정상/기타→**상세**. 잘못된 상태에 위험 액션을 노출하지 않는다. |
| **kpi-filter-chip** | 요약 헤더 | 클릭 = 해당 심각도로 목록 필터(단일 선택, `전체`로 해제). aria-pressed 반영. |
| **filter-chip / 검색** | 워크벤치 툴바 | 플랫폼(배민/쿠팡)·인증필요 토글(다중) + 이름/센터 검색(타이핑 즉시 필터) + 정렬 셀렉트 + 표시 카운트("8 / 100곳"). |
| **target-drawer** | 모니터링·관리 | 탭 `상세`/`편집`. 상세 = 왜→사실(dl)→이력(타임라인)→조치, 행 `data-*`로 즉시 렌더(서버 왕복 0). 편집 = **'편집' 탭을 처음 열 때 lazy fetch** 로 전체 현재값(표시명·센터·external_id·url·주기) 로드 → 같은 드로어에서 수정(연결은 드롭다운, 부분 갱신 "비우면 유지"). **이 드로어 편집이 업체 편집의 정본 경로** — 관리 모드의 인라인 edit-row 는 격하/제거. Esc/스크림 클릭/✕ 로 닫기. |
| **고급 운영 액션** | 관리 ▸ 고급 운영 액션 | 구독 중지/재개·dispatch 폐기·job 재시도·agent 배정. 대상은 **전부 드롭다운**이어야 한다(내부 UUID 직접 입력 금지). ⚠️ **현재 코드는 raw text input**(`_actions.html`). 드롭다운화 시 구독 옵션은 기존 `/admin/subscriptions/options` 재사용 가능하나 **job·agent·dispatch 옵션 엔드포인트는 없어 신규 필요**(target 옵션은 기존 재사용). 파괴적 액션은 이름 박은 확인. |
| **guided-form** | 관리 ▸ 새 업체 추가 | **한 화면 번호 2-섹션**(① 업체 만들기 → ② 채널 연결). 위저드(멀티페이지) 아님 — HTMX stateless POST 라 스텝 상태보존을 만들지 않는다. ① = 계정 드롭다운(첫 필드) + 이름·센터·url·external_id·주기, `POST /admin/monitoring-targets`. ② = 채널 드롭다운 + "변경시에만 전송", `POST /admin/delivery-rules`. **① 제출 성공 시 방금 만든 업체를 ②에 선택된 상태로 자동 주입**(② 헤드라인 "○○에 채널 연결")해 "같은 업체 다시 찾기"를 없앤다. **② 는 선택**(채널 미정 업체는 채널 없이 등록 허용, "나중에" 가능). 계정·채널은 **드롭다운**(목록에 없으면 드롭다운 맨 끝 "＋ 새 …만들기" → **인라인 미니폼**, 보조 섹션으로 점프 금지). **이름·계정 미입력 시 제출 차단**(required input 까지 버튼 비활성 가드 + 비활성 사유 마이크로카피). |
| **collapsible (`details`)** | Agent fleet · 채널 · 보조 관리 | 기본 접힘 — 트리아지를 가리지 않는다. summary 에 핵심 요약(예: "Agent 2/2 온라인"). |

## State Patterns

| 상태 | 서피스 | 처리 |
|---|---|---|
| 전부 정상 | 모니터링 목록 | 빈 표 금지 → "🟢 {N}곳 모두 정상 · 지금 조치할 항목 없음" 안심 메시지. |
| 필터 결과 0 | 워크벤치 | "조건에 맞는 업체가 없습니다 — 필터를 조정하세요" + 필터 초기화 버튼. |
| 첫 로딩 | 목록/드로어 | 스켈레톤 행(레이아웃 유지, 깜빡임 방지). |
| **30초 갱신 실패** | 앱바 + 영향 영역 | 조용한 실패 금지 → "갱신 실패 · 마지막 HH:MM · 재시도 중" + stale 배지. HTMX 부분 swap 실패를 감지해 라이브 신뢰를 지킨다. |
| 액션 진행 | 버튼/행 | 버튼 인라인 비활성+스피너. 동안 행 dim(`htmx-swapping`). |
| 액션 결과 | 해당 행/드로어 근처 | 성공/실패를 **맥락 근처**에 표시(멀리 있는 전역 토스트 의존 금지). `role="status"`. |
| 위험 액션 확인 | 모든 파괴적 액션 | 대상 이름 박은 확인 → "정말 '{업체명}'을 {동작}할까요?". 폐기/중지/비활성 공통. |
| 인증 필요(secret) | 대상/드로어 | 사유는 사람 문장 + 머신코드 괄호. 인증번호·토큰·비밀번호 **절대 비노출**. 자동복구 실패 반복 시 중지 정책 유지. |
| **가이드 폼 단계 진행** | 새 업체 추가 ▸ 진행 배지 | ① 미완료 = 무채 번호 배지(`{colors.border}`/`{colors.muted-foreground}`), 현재 = 골든 ring(`{colors.ring}`), 완료 = `{colors.ok}` 체크. ① 제출 성공 시 ① 배지 체크 전환 + ② 현재로 승격. **색 단독 금지**(번호+색+체크 3중). 모바일은 "1/2 단계" 텍스트 폴백. |
| **가이드 폼 완료** | 새 업체 추가 ▸ 결과 | 인라인 span 금지 → 확인 카드: "✓ {업체명} 추가됨 · 다음 주기부터 수집"(`{colors.ok}`). 쿠팡 센터명 경고 있으면 카드 안 crit 줄 + **[센터명 설정]**. 다음 행동 2갈래: **[계속 추가]**(무채, 폼 리셋·① 복귀) / **[모니터링으로 가기]**(골든 1차, `HX-Redirect`). |
| 쿠팡 센터명 미설정 | 새 업체 추가 ▸ 입력/제출 | **소프트 게이트(차단 아님).** 사전(목표·미구현) = COUPANG 계정 선택 시 센터/상점명 라벨에 "쿠팡 필수" crit 강조 + "비우면 오발송 위험" 도움말. ⚠️ **현재 코드 미구현** — 계정 옵션 fragment(`platform_account_options`)가 platform 정보를 `data-platform`으로 싣지 않고 라벨 문자열에만 담으며, 계정 select 에 onchange 핸들러가 없다. 구현하려면 옵션에 platform 을 `data-*`로 노출 + onchange 감지 필요(채널 옵션은 이미 `data` 동봉 — 같은 패턴 적용). 사후(구현됨) = `center_name_risky` crit 경고 + [센터명 설정]. "센터명 아직 모름" 등록은 막지 않되 경고로 안전망. |
| 외부 식별자 입력 | 채널 생성/편집 (chat_id 등) | 라벨 아래 **출처 안내**("텔레그램 그룹에서 …로 확인") + 형식 검증(chat_id 음수/숫자, 빈 thread_id 허용). 잘못된 형식은 제출 전 인라인 피드백. 드롭다운으로 만들지 않는다(외부 발급값). |
| secret ref 입력 | 계정 생성/편집 (vault:// 등) | "**평문 금지 · 핸들만(vault://…)**" 안내 + prefix(`vault:`/`local:`/`env:`/`dpapi:`) 즉시 검증. 평문 입력은 fail-closed(서버 400 유지). bot token/webhook secret 은 password 입력 + "설정됨/미설정" 라벨, 값 화면 표시 절대 금지. |

## Interaction Primitives

**클릭·검색 우선(비개발 운영자).** 입력은 본질에 따라 셋으로 나뉜다 — 무엇을 가리키든 가능하면 검색·드롭다운·행 클릭으로, 불가피한 타이핑은 출처 안내·형식 검증으로 안전하게.

- **내부 엔티티 참조 → raw UUID 입력 금지.** 우리 DB의 업체/계정/채널/구독/job/agent/dispatch 는 전부 대시보드에 이미 존재한다. 이들을 가리킬 땐 검색·드롭다운·행 클릭만 쓴다(고급 운영 액션의 subscription/dispatch/job/target/agent id 포함). UUID 타이핑은 어디서도 요구하지 않는다.
- **외부 발급 식별자(telegram chat_id·thread_id·kakao 방명) → 타이핑 불가피, 안내로 보완.** 텔레그램·카카오가 발급하는 값이라 우리 목록에 미리 없다. 드롭다운으로 못 만든다 — 대신 입력란에 **"이 값을 어디서 얻는지" 인라인 안내 + 형식 검증·붙여넣기 도우미**(예: chat_id 음수/숫자 즉시 검증)를 둔다.
- **secret ref 핸들(`vault://…` 등·bot token) → 핸들/마스킹만, 안내로 보완.** 외부 Secrets Manager 핸들이라 드롭다운 대상이 아니다. 입력란에 **"평문 금지 · 핸들만(vault://…)" 안내 + prefix 형식 즉시 검증**. token/webhook secret 은 password 입력 + "설정됨/미설정" 라벨로만(평문 절대 비노출, 기존 정책).

- **행 클릭 / Enter** → 업체 상세 드로어.
- **검색 우선** — 100개는 외우는 게 아니라 찾는다. 이름/센터 검색이 모든 목록의 기본 진입.
- **상황맞춤 액션** — 행마다 "지금 할 수 있는 1가지"를 강조. 나머지는 드로어로.
- **목록에 없으면 그 자리에서 만들기** — 계정/채널 드롭다운 맨 끝 "＋ 새 …만들기" sentinel → **같은 섹션 인라인 미니폼**으로 생성, 생성 즉시 그 항목이 선택됨. 보조 섹션으로 점프해 흐름을 끊지 않는다.
- **이름 박은 위험 확인** — 100개 중 오조작 방지의 핵심.
- **Esc** — 드로어/팝오버 닫기. 스크림 클릭도 닫기.
- **HTMX polling 30초** — 대상·Agent·채널·인증필요. 갱신 시각을 앱바에 stamp. swap 실패는 가시화(위 상태표).
- **금지:** 무한 스크롤(필터·페이지로), 호버 전용 액션(모바일 탭 가능해야), 모달 2단 중첩, raw UUID 입력.

## Accessibility Floor

행동. 시각 대비는 `DESIGN.md`(의미색 모드별 보정, AA 목표).

- **WCAG 2.2 AA** 목표. 의미색 대비는 라이트/다크 각각 보정값 유지.
- **색 단독 금지** — 심각도 = 색 + 점 + 한글 라벨, 신선도 = 색 + "47분 전" 텍스트.
- **키보드** — 행 포커스/Enter로 드로어, **Esc 닫기**, 드로어 내 포커스 트랩, 보이는 포커스 링(`{colors.ring}`).
- **`aria-live`** — 갱신·액션 결과를 스크린리더에 통지. 모드 전환 시 현재 모드 안내.
- **모바일** — 터치 타깃 ≥ `{spacing.touch-min}`(44px). `prefers-reduced-motion` 존중(펄스/트랜지션 차단 — 기존 유지).
- **폼** — 라벨-필드 연결, 필수 표시, 에러를 필드 근처에.

## Key Flows

### J1 — 트리아지 (핵심·매일 #1) · 주인공 민지

1. 아침, 민지가 대시보드를 연다. 요약 헤더: **위험 3 · 중지 1**이 빨강으로 떠 있다.
2. 목록은 이미 위험순. 맨 위 **'동대문엽기떡볶이 본점'** 행에 *"로그인 만료 · 인증 확인 필요 / 수집 47분 전(빨강) · 전송 1시간 전 · 주기 5분"* 과 골든 **[인증 확인]** 버튼. 신선도 줄에서 민지는 "수집도 전송도 멈췄고, 5분마다 돌아야 할 게 47분째 막혔다"를 한눈에 읽는다.
3. 민지가 행을 클릭 → 우측 드로어. **왜**(로그인 세션 만료, 인증하면 다음 주기 자동 복구) → 사실(센터·주기·마지막 수집/전송) → 최근 이력(14:31 수집 실패 ×2, 13:45 전송 성공).
4. **(climax)** 민지가 드로어의 **[인증 확인]** 을 누른다. 버튼이 잠깐 스피너 → *"인증 확인 요청됨 · 다음 주기에 재시도"* 가 드로어 안에 뜬다. 위험 카운트가 3→2로 줄고, 그 행이 조용히 목록 위에서 내려간다. **"내가 모르는 사이 멈춰 있던 가게"가 사라졌다.**
5. 남은 위험 2건도 같은 리듬으로 처리. 92곳 정상은 안 건드린다.

### J2 — 신규 등록 · 민지

1. 새 가맹점이 들어왔다. 민지가 **[관리]** 탭으로. **[+ 새 업체 추가]** → 가이드 폼이 열린다. **① 업체 만들기** 섹션 번호가 골든으로 켜져 "여기부터"를 알려준다.
2. **① 업체 만들기.** 계정 드롭다운에서 "배민 · 본사계정" 선택(목록에 없었다면 "＋ 새 계정 만들기"로 그 자리에서 생성) → 이름·센터·url·주기 입력. 이름과 계정을 채우자 **[업체 추가]** 버튼이 골든으로 활성화된다.
3. **[업체 추가]**. ① 배지가 녹색 체크로 바뀌고 **② 채널 연결** 이 골든으로 승격된다. ② 헤드라인이 *"○○치킨 본점에 채널 연결"* — 민지가 **방금 만든 업체를 다시 고를 필요가 없다**(자동 주입).
4. **② 채널 연결.** 채널 "텔레그램 단체방" 드롭다운에서 선택 + "변경시에만 전송" 체크. (채널이 아직 없으면 **[나중에]** 로 넘어가도 된다.)
5. **(climax)** **[연결하기]**. 확인 카드가 뜬다: *"✓ ○○치킨 본점 추가됨 · 다음 주기부터 수집"*. **id를 한 번도, raw 문자열을 한 글자도 입력하지 않았다.** 카드의 **[계속 추가]** 로 다음 가맹점을 같은 리듬으로, 또는 **[모니터링으로 가기]** 로 복귀.
6. 모니터링 탭으로 돌아가면 새 업체가 목록에 정상으로 보인다.

> **[NOTE FOR DEV]** ①과 ②는 백엔드 라우트가 분리(`POST /admin/monitoring-targets` + `POST /admin/delivery-rules`, delivery-rule 은 target 커밋 후 가능)되어 **2-POST**다. "한 흐름" 체감은 ① 성공 시 만든 업체를 ②에 자동 주입(`HX-Trigger` 에 target id/name 동봉)해 만든다. 진정한 single-submit(target+rule 트랜잭션) 통합 라우트는 향후 dev 스토리 후보.

### J3 — 온콜 모바일 · 민지

1. 저녁, 자리를 비운 민지의 폰에 **텔레그램 알림**: *"[위험] 강남교자 2호점 수집 3회 실패 — 확인"* + 링크. *(알림 발송은 백엔드 의존 — 아래 NOTE)*
2. 링크 탭 → 모바일 브라우저가 **그 업체 상세 드로어(전체폭)** 로 바로 연다(딥링크).
3. **(climax)** 사유 *"Gmail 2FA 인증 필요"* 를 읽고 **[인증 확인]** 한 번 탭. 끝. 데스크톱 앞으로 갈 필요 없이, 9열 표를 헤맬 필요 없이.

> **[NOTE FOR DEV]** J3의 진입점인 "멈추면 텔레그램 알림"은 **백엔드 의존성**이다(심각도 전이 시 push). UX 범위는 (1) 딥링크 가능한 안정 URL의 업체 상세, (2) 모바일 전체폭 드로어까지. 실제 알림 발송 트리거/대상/채널 설정은 별도 dev 스토리로 스코핑 필요(알림 설정 UI 위치 미정 — 관리 탭 후보).

## Inspiration & Anti-patterns

- **계승 (지킨 것):** darkmatter 토큰·다크 우선·모노 타이포; 심각도 우선 자동 정렬; HTMX 30초 라이브 + 갱신 시각 stamp; 읽기/쓰기 분리 + audit; secret 핸들 정책.
- **거부 — raw-id 수동 액션 패널:** 구독/dispatch/job/agent를 **내부 UUID 텍스트**로 조작하던 벽을 제거. 행 인라인 액션 + **드롭다운**(내부 엔티티는 우리 목록에 있음)으로 대체(비개발 운영자 부적합). 단 **외부 발급 식별자(chat_id 등)·secret ref 핸들은 제거 대상이 아니다** — 본질상 타이핑이 불가피하므로 출처 안내 + 형식 검증으로 안전하게 만든다(거짓 완결 금지).
- **거부 — 인증필요 별도 표:** 대상 목록 필터/배지로 흡수(한 곳에서 본다).
- **거부 — 절대시각 문자열:** 상대시간 + 신선도색으로 대체.
- **거부 — 관측과 변경의 한 벽 혼재:** 모니터링(트리아지)과 관리(CRUD)를 모드로 분리.
- **거부 — 익명 confirm:** 대상 이름 박은 확인으로 대체.

## 보안·비노출 규칙 (invented — 도메인 특수)

- secret/토큰/OTP/비밀번호는 화면·메시지·이력·딥링크 어디에도 표시하지 않는다. 계정 자격은 핸들(`*_ref`)만.
- secret 입력란은 **핸들만 받는다** — "평문 금지 · vault://… 핸들만" 안내 + prefix(`vault:`/`local:`/`env:`/`dpapi:`) 즉시 검증. 평문 입력은 서버에서 400(fail-closed)으로 거부하고, 이 거부 사유를 사람 문장으로 알린다. bot token/webhook secret 은 입력 후 값이 아닌 "설정됨/미설정"만 보인다.
- 쿠팡 기대 센터/상점명 미설정·배민 기본값 잔존은 **오발송 위험**으로 위험(crit) 취급하고, 행/드로어에서 명시 경고 + **[센터명 설정]** 액션을 띄운다.
- 인증 자동복구 실패가 반복되면 반복 인증 요청 대신 **탭/대상 중지** 정책을 UI도 따른다(잘못된 실적 발송 방지 우선).

## 규모 100+ 대상 처리 (invented — 핵심 제약)

- 목록은 **항상 검색·필터·카운트**를 동반한다(이름/센터 검색, 심각도·플랫폼·인증필요 필터, "N / 100곳" 표시).
- 기본 정렬 = 심각도 위험순. "지금 조치할 N곳"을 먼저, 정상 다수는 접거나 필터 뒤로.
- 무한 스크롤 대신 페이지/가상 스크롤. 100곳이 늘어도 트리아지 속도가 유지되어야 한다.
