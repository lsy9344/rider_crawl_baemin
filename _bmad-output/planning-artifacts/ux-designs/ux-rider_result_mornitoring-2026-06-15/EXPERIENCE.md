# rider_server 운영 대시보드 — Experience Spine (EXPERIENCE.md)

> 동작·정보구조·상태·인터랙션·접근성·여정의 정본. 시각 정체성은 `DESIGN.md`. 어떤 목업과 충돌해도 두 스파인이 이긴다.
> 시각 참조 목업: [`mockups/admin-redesign.html`](./mockups/admin-redesign.html) — 모니터링+관리 양 모드 클릭 가능 프로토타입.

## Foundation

- **폼팩터:** 반응형 웹. **데스크톱 : 모바일 ≈ 5:5**, 둘 다 1급. 벽걸이/대형 모니터 전용 모드는 없음.
- **UI 시스템:** 자체 완결형 — **darkmatter 토큰(인라인 CSS 변수) + Jinja2 + HTMX 2(CDN, 빌드 0)** 서버 렌더. 외부 컴포넌트 라이브러리 없음. `DESIGN.md` 가 시각 정본.
- **테넌시:** **단일 운영자/단일 고객.** 100여 개 업체(모니터링 대상)가 한 tenant 밑에 달림. tenant 전환 UI 불필요(`?tenant=` 는 내부 seam 으로만 잔존). 화면은 "업체 100개를 잘 보여주기"에 100% 집중.
- **쓰기 안전:** 대시보드 읽기는 read-only(`DashboardService`), 상태 변경은 별도 action/CRUD 라우터(OPERATOR↑ 권한 + actor·시각·diff audit). UI 재설계가 이 경계를 바꾸지 않는다.

## Information Architecture

상단 고정 바(앱바 + **모드 탭**) 아래 두 모드. 인증필요·수동액션 패널 같은 옛 별도 섹션은 **대상 워크벤치로 흡수**됐다.

| 서피스 | 진입 | 목적 |
|---|---|---|
| **모니터링** (기본) | 앱 열기 / 모드탭 | J1 트리아지. 요약(KPI 필터) + 대상 워크벤치 + (접힘) Agent·채널 |
| **관리** | 모드탭 | J2 등록/편집. 검색형 업체 목록 + `+ 새 업체 추가` 가이드 폼 + (접힘) 계정·채널·템플릿 |
| **업체 상세 드로어** | 어느 목록이든 행 클릭 / 텔레그램 알림 딥링크 | 한 업체의 왜·사실·이력·조치. 탭 `상세`/`편집`. 모니터링·관리 공용 |
| **새 업체 추가** | 관리 ▸ `+ 새 업체 추가` | 가이드 단일 폼(계정→정보→채널, 전부 드롭다운) |

- **요약 KPI = 필터.** 정상/주의/위험/중지 칩이 곧 대상 목록 필터. 숫자→항목 단절 해소.
- **딥링크 대상 상세.** `/admin/t/{target}` 류 안정 URL로 텔레그램 알림에서 바로 진입(모바일 1급). → 합성 참조: [`mockups/admin-redesign.html`](./mockups/admin-redesign.html). 스파인이 충돌 시 이김.

## Voice and Tone

마이크로카피. 브랜드 보이스/미감은 `DESIGN.md.Brand & Style`.

| Do | Don't |
|---|---|
| "지금 조치할 8곳" | "조치 필요 항목: 8" |
| "로그인 만료 · 인증 확인 필요" | "AUTH_REQUIRED" / "MissingPerformanceDataError" |
| "🟢 100곳 모두 정상 · 지금 조치할 항목 없음" | (빈 표) |
| "정말 '동대문엽기떡볶이 본점'을 비활성화할까요?" | "비활성화하시겠습니까?" |
| "갱신 실패 · 마지막 14:30 · 재시도 중" | (조용히 멈춘 화면) |
| 운영자 언어: "지금 수집" / "미리보기" | 개발자 용어: "test-crawl" / "dry-run" |

원칙: 사람 문장 우선, 기계 코드는 숨기되 필요 시 괄호 보조. 비난조 금지·행동 유도. **secret/토큰/OTP 는 어떤 화면·메시지·이력에도 표시하지 않는다(핸들만).**

## Component Patterns

행동 명세. 시각은 `DESIGN.md.Components`.

| 컴포넌트 | 사용처 | 행동 규칙 |
|---|---|---|
| **target-row** | 모니터링·관리 목록 | 행 전체 클릭 → 상세 드로어. 상황맞춤 1차 액션 버튼은 `event.stopPropagation`(행 클릭과 분리). 심각도 내림차순 자동 정렬(위험·중지 위로). **데스크톱 열은 고정폭 트랙(플랫폼·심각도 배지·액션 열)을 공유**해 행마다 정렬이 맞는다 — 글자수에 따라 어긋나면 안 된다(배지/액션 열에 `auto`/content-기반 폭 금지, 텍스트 열만 `minmax(0,fr)`로 가변+말줄임). |
| **상황맞춤 액션** | target-row · 드로어 | 상태→액션 매핑: 로그인만료/AUTH→**인증 확인** · 쿠팡 센터명 미설정→**센터명 설정** · 연속 수집실패→**인증 확인**+**재시도** · 중지→**활성화** · 수집지연→**지금 수집** · 정상/기타→**상세**. 잘못된 상태에 위험 액션을 노출하지 않는다. |
| **kpi-filter-chip** | 요약 헤더 | 클릭 = 해당 심각도로 목록 필터(단일 선택, `전체`로 해제). aria-pressed 반영. |
| **filter-chip / 검색** | 워크벤치 툴바 | 플랫폼(배민/쿠팡)·인증필요 토글(다중) + 이름/센터 검색(타이핑 즉시 필터) + 정렬 셀렉트 + 표시 카운트("8 / 100곳"). |
| **target-drawer** | 모니터링·관리 | 탭 `상세`/`편집`. 상세 = 왜→사실(dl)→이력(타임라인)→조치. 편집 = 같은 드로어에서 필드 수정(연결은 드롭다운). Esc/스크림 클릭/✕ 로 닫기. |
| **guided-form** | 관리 ▸ 새 업체 추가 | 번호 스텝. 계정·채널·템플릿은 **드롭다운**(목록에 없으면 "＋ 새 …만들기"). 필수 필드(이름) 미입력 시 제출 차단. |
| **collapsible (`details`)** | Agent fleet · 채널 · 보조 관리 | 기본 접힘 — 트리아지를 가리지 않는다. summary 에 핵심 요약(예: "Agent 2/2 온라인"). |

## State Patterns

| 상태 | 서피스 | 처리 |
|---|---|---|
| 전부 정상 | 모니터링 목록 | 빈 표 금지 → "🟢 {N}곳 모두 정상 · 지금 조치할 항목 없음" 안심 메시지. |
| 필터 결과 0 | 워크벤치 | "조건에 맞는 업체가 없습니다 — 필터를 조정하세요" + 필터 초기화 버튼. |
| 첫 로딩 | 목록/드로어 | 스켈레톤 행(레이아웃 유지, 깜빡임 방지). |
| **30초 갱신 실패** | 앱바 + 영향 영역 | 조용한 실패 금지 → "갱신 실패 · 마지막 HH:MM · 재시도 중" + stale 배지. HTMX 부분 swap 실패를 감지해 라이브 신뢰를 지킨다. |
| 액션 진행 | 버튼/행 | 버튼 인라인 비활성+스피너. 동안 행 dim(`htmx-swapping`). |
| 액션 결과 | 해당 행/드로어 근처 | 성공/실패를 **맥락 근처**에 표시(멀리 있는 전역 토스트 의존 금지). `role="status"`. |
| 위험 액션 확인 | 모든 파괴적 액션 | 대상 이름 박은 확인 → "정말 '{업체명}'을 {동작}할까요?". 폐기/중지/비활성 공통. |
| 인증 필요(secret) | 대상/드로어 | 사유는 사람 문장 + 머신코드 괄호. 인증번호·토큰·비밀번호 **절대 비노출**. 자동복구 실패 반복 시 중지 정책 유지. |

## Interaction Primitives

**클릭·검색 우선(비개발 운영자).** raw id 타이핑은 전 화면에서 제거됐다 — 무엇을 가리키든 검색·드롭다운·행 클릭으로 한다.

- **행 클릭 / Enter** → 업체 상세 드로어.
- **검색 우선** — 100개는 외우는 게 아니라 찾는다. 이름/센터 검색이 모든 목록의 기본 진입.
- **상황맞춤 액션** — 행마다 "지금 할 수 있는 1가지"를 강조. 나머지는 드로어로.
- **이름 박은 위험 확인** — 100개 중 오조작 방지의 핵심.
- **Esc** — 드로어/팝오버 닫기. 스크림 클릭도 닫기.
- **HTMX polling 30초** — 대상·Agent·채널·인증필요. 갱신 시각을 앱바에 stamp. swap 실패는 가시화(위 상태표).
- **금지:** 무한 스크롤(필터·페이지로), 호버 전용 액션(모바일 탭 가능해야), 모달 2단 중첩, raw UUID 입력.

## Accessibility Floor

행동. 시각 대비는 `DESIGN.md`(의미색 모드별 보정, AA 목표).

- **WCAG 2.2 AA** 목표. 의미색 대비는 라이트/다크 각각 보정값 유지.
- **색 단독 금지** — 심각도 = 색 + 점 + 한글 라벨, 신선도 = 색 + "47분 전" 텍스트.
- **키보드** — 행 포커스/Enter로 드로어, **Esc 닫기**, 드로어 내 포커스 트랩, 보이는 포커스 링(`{colors.ring}`).
- **`aria-live`** — 갱신·액션 결과를 스크린리더에 통지. 모드 전환 시 현재 모드 안내.
- **모바일** — 터치 타깃 ≥ `{spacing.touch-min}`(44px). `prefers-reduced-motion` 존중(펄스/트랜지션 차단 — 기존 유지).
- **폼** — 라벨-필드 연결, 필수 표시, 에러를 필드 근처에.

## Key Flows

### J1 — 트리아지 (핵심·매일 #1) · 주인공 민지

1. 아침, 민지가 대시보드를 연다. 요약 헤더: **위험 3 · 중지 1**이 빨강으로 떠 있다.
2. 목록은 이미 위험순. 맨 위 **'동대문엽기떡볶이 본점'** 행에 *"로그인 만료 · 인증 확인 필요 / 수집 47분 전(빨강)"* 과 골든 **[인증 확인]** 버튼.
3. 민지가 행을 클릭 → 우측 드로어. **왜**(로그인 세션 만료, 인증하면 다음 주기 자동 복구) → 사실(센터·주기·마지막 수집/전송) → 최근 이력(14:31 수집 실패 ×2, 13:45 전송 성공).
4. **(climax)** 민지가 드로어의 **[인증 확인]** 을 누른다. 버튼이 잠깐 스피너 → *"인증 확인 요청됨 · 다음 주기에 재시도"* 가 드로어 안에 뜬다. 위험 카운트가 3→2로 줄고, 그 행이 조용히 목록 위에서 내려간다. **"내가 모르는 사이 멈춰 있던 가게"가 사라졌다.**
5. 남은 위험 2건도 같은 리듬으로 처리. 92곳 정상은 안 건드린다.

### J2 — 신규 등록 · 민지

1. 새 가맹점이 들어왔다. 민지가 **[관리]** 탭으로.
2. **[+ 새 업체 추가]** → 가이드 폼. ① 계정 드롭다운에서 "배민 · 본사계정" 선택 ② 이름·센터·url·주기 입력 ③ 채널 "텔레그램 단체방" + 템플릿 선택 + "변경시에만 전송" 체크.
3. **(climax)** **[업체 추가 ✓]**. 폼이 *"○○치킨 본점 추가됨 · 다음 주기부터 수집"* 으로 확인. id를 한 번도 입력하지 않았다.
4. 모니터링 탭으로 돌아가면 새 업체가 목록에 정상으로 보인다.

### J3 — 온콜 모바일 · 민지

1. 저녁, 자리를 비운 민지의 폰에 **텔레그램 알림**: *"[위험] 강남교자 2호점 수집 3회 실패 — 확인"* + 링크. *(알림 발송은 백엔드 의존 — 아래 NOTE)*
2. 링크 탭 → 모바일 브라우저가 **그 업체 상세 드로어(전체폭)** 로 바로 연다(딥링크).
3. **(climax)** 사유 *"Gmail 2FA 인증 필요"* 를 읽고 **[인증 확인]** 한 번 탭. 끝. 데스크톱 앞으로 갈 필요 없이, 9열 표를 헤맬 필요 없이.

> **[NOTE FOR DEV]** J3의 진입점인 "멈추면 텔레그램 알림"은 **백엔드 의존성**이다(심각도 전이 시 push). UX 범위는 (1) 딥링크 가능한 안정 URL의 업체 상세, (2) 모바일 전체폭 드로어까지. 실제 알림 발송 트리거/대상/채널 설정은 별도 dev 스토리로 스코핑 필요(알림 설정 UI 위치 미정 — 관리 탭 후보).

## Responsive & Platform

| 브레이크포인트 | 동작 |
|---|---|
| 데스크톱 (> 720px) | target-row = 다열 그리드. 상세 = 우측 슬라이드 드로어(440px, 목록 맥락 유지). |
| 모바일 (≤ 720px) | target-row → **카드**(심각도 바 + 이름/플랫폼 + 사유줄 + 액션 아래로 배치). 드로어 = 전체폭. 툴바 wrap. 모드탭 유지. 터치 타깃 ≥ 44px. |

read + 핵심 1탭 조치는 모바일에서 완결되어야 한다(J3). 대량 등록·복잡 편집의 주 무대는 데스크톱.

## Inspiration & Anti-patterns

- **계승 (지킨 것):** darkmatter 토큰·다크 우선·모노 타이포; 심각도 우선 자동 정렬; HTMX 30초 라이브 + 갱신 시각 stamp; 읽기/쓰기 분리 + audit; secret 핸들 정책.
- **거부 — raw-id 수동 액션 패널:** 구독/dispatch/job/agent를 UUID 텍스트로 조작하던 벽을 제거. 행 인라인 액션 + 검색 선택으로 대체(비개발 운영자 부적합).
- **거부 — 인증필요 별도 표:** 대상 목록 필터/배지로 흡수(한 곳에서 본다).
- **거부 — 절대시각 문자열:** 상대시간 + 신선도색으로 대체.
- **거부 — 관측과 변경의 한 벽 혼재:** 모니터링(트리아지)과 관리(CRUD)를 모드로 분리.
- **거부 — 익명 confirm:** 대상 이름 박은 확인으로 대체.

## 보안·비노출 규칙 (invented — 도메인 특수)

- secret/토큰/OTP/비밀번호는 화면·메시지·이력·딥링크 어디에도 표시하지 않는다. 계정 자격은 핸들(`*_ref`)만.
- 쿠팡 기대 센터/상점명 미설정·배민 기본값 잔존은 **오발송 위험**으로 위험(crit) 취급하고, 행/드로어에서 명시 경고 + **[센터명 설정]** 액션을 띄운다.
- 인증 자동복구 실패가 반복되면 반복 인증 요청 대신 **탭/대상 중지** 정책을 UI도 따른다(잘못된 실적 발송 방지 우선).

## 규모 100+ 대상 처리 (invented — 핵심 제약)

- 목록은 **항상 검색·필터·카운트**를 동반한다(이름/센터 검색, 심각도·플랫폼·인증필요 필터, "N / 100곳" 표시).
- 기본 정렬 = 심각도 위험순. "지금 조치할 N곳"을 먼저, 정상 다수는 접거나 필터 뒤로.
- 무한 스크롤 대신 페이지/가상 스크롤. 100곳이 늘어도 트리아지 속도가 유지되어야 한다.
