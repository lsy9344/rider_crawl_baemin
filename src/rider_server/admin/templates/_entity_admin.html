{# Story 5.11 엔티티 관리(생성/편집/비활성화) 폼 섹션 — Jinja2+HTMX(신규 npm/빌드 0, HTMX 2 CDN). #} {# 폼은 정적, 목록만 30초 polling. secret 입력은 *_ref 핸들만(평문 금지 — 라벨로 명시, autoescape 유지). #} {# 생성 폼은 HTMX 가 입력을 urlencoded 로 직렬화(hx-post). 편집/비활성화는 id 가 URL path 라 JS 로 조립. #}

엔티티 CRUD — 생성/편집/비활성화는 OPERATOR↑ 권한이 필요하며, actor/시각/변경 diff 가 audit 에 기록됩니다. 비활성화는 물리 삭제가 아니라 soft delete(상태 전이)입니다.

엔티티 액션 결과가 여기에 표시됩니다.
고객(Tenant)
플랫폼 계정 (secret 은 *_ref 핸들만 — 평문 금지)
모니터링 대상 (쿠팡 center_name 비었거나 배민 기본값이면 위험 경고)
메시지 채널 (chat_id/thread_id/방명 = 라우팅 식별자, secret 아님)
