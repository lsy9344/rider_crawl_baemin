# 새 고객 Admin 세팅 순서

운영자가 Admin(`/admin`)에서 새 고객을 전송 가능 상태로 만드는 표준 순서다. 비밀번호·토큰·OTP·webhook secret 같은 비밀값은 이 문서에 적지 않는다(Admin은 비밀값을 핸들로만 저장한다).

## 순서

1. 고객 생성 또는 선택
2. 플랫폼 계정 생성(배민/쿠팡)
3. 모니터링 대상 생성(업체/센터)
4. 메시지 채널 생성(텔레그램 chat/topic 또는 카카오 방)
5. 채널 전송 테스트(채널이 실제로 도착하는지 확인)
6. **전송 규칙 생성: 대상 → 채널 연결**
7. 수집 테스트
8. 고객 실전송 ON(`실제 메시지 보내기`)

6번이 빠지면 등록된 설정에서 `고객 전송 ON` 인데 `메신저 —`로 보이고, 실제로는 전송되지 않는다.

## 채널 생성만으로 자동 전송되지 않는다

- **메시지 채널** = 보낼 수 있는 목적지(주소록).
- **전송 규칙(DeliveryRule)** = 어떤 모니터링 대상 결과를 어느 채널로 보낼지의 연결.
- 채널을 만들거나 전송 테스트를 통과해도, 대상 ↔ 채널 **연결(전송 규칙)** 을 만들기 전에는 그 대상의 실적이 전송되지 않는다.

이건 의도된 안전장치다(fail-closed). 한 고객 안에 대상이 여럿일 수 있고, 새 채널을 테스트하려고 만들었는데 기존 모든 대상이 그 방으로 나가면 오발송이다. 연결 규칙이 없으면 "보내지 않음"이 맞다.

## 등록된 설정 화면 읽는 법

- `고객 전송`: 고객의 실전송 상위 토글(`tenant.sending_enabled`).
- `대상 연결`:
  - `ON`: 고객 전송 ON + 대상에 활성 전송 규칙 있음(전송 준비 완료).
  - `연결 필요`: 고객 전송 ON 이지만 활성 전송 규칙이 없음 → 관리 탭에서 전송 규칙을 만들어야 한다.
  - `OFF`: 고객 전송 토글이 OFF.
- `메신저`에 `—`와 `전송 규칙 없음`이 같이 보이면 그 대상은 아직 어느 채널과도 연결되지 않았다는 뜻이다.

대상 1개 + 활성 채널 1개인 고객은 관리 탭 `전송 연결` 섹션에서 `이 대상에 채널 연결` 빠른 연결 버튼으로 바로 규칙을 만들 수 있다.

## 표시는 readiness, 실전송 게이트는 별도

등록된 설정의 `대상 연결`은 운영 UI용 준비 상태 표시다. 실제 전송은 runtime의 최종 게이트를 계속 따른다: 고객 전송 토글, 전역 발송 게이트, 전송 허용 시간창, 활성 전송 규칙, 활성 채널, 메시지 변경 조건, dispatch worker/agent 상태.
