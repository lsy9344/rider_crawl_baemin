# Runbook: KakaoTalk 모호한 채팅방 (Story 5.9 / AC3, NFR-15·17)

> 장애 분류(`FailureCategory`): `KAKAO_FAILURE`. 관련 지표: `kakao_queue_lag_seconds`
> (모호건 보류가 FIFO 큐를 적체시킬 수 있음 — `queue_lag.md` 연계).

## 증상

- Kakao 전송 대상 채팅방이 **동명/모호**해 정확히 어느 방인지 확정할 수 없음 → 전송 보류
  (**미발송**), `KAKAO_FAILURE` 로 분류.
- 단일 FIFO 세션이라 선두 모호건이 막히면 뒤 전송이 적체(`queue_lag` 알림으로 번질 수 있음).

## 원인

- 같은 이름의 채팅방이 둘 이상 존재(동명 방).
- 채팅방 이름 변경/재생성으로 등록된 방명과 현재 방 목록이 불일치(Story 4.6 정확한 채팅방 검증).

## 조치

1. **임의 창(채팅방) 전송 절대 금지** — 잘못된 방에 고객 실적이 나가는 오발송이 적체보다
   위험하다. 모호하면 보내지 않는다(fail-closed, `KAKAO_FAILURE`).
2. 대상의 정확한 채팅방을 식별 가능한 키(방명 + 추가 식별자)로 확정하도록 등록 정보를 정정.
3. 동명 방이 불가피하면 방 이름을 고유하게 변경하거나 식별 가능한 방으로 재지정.
4. 확정 후 보류 해소 → 큐 적체(`kakao_queue_lag_seconds`)가 회복되는지 확인(`queue_lag.md`).

## 에스컬레이션

- 고객이 방 정리를 못 해 반복 모호 → 고객 커뮤니케이션 담당이 방 정비 안내.
- 적체가 SLA 를 위협하면 운영 책임자에게 보고(미발송 유지 전제).
