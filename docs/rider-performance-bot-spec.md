## 1. 목적

쿠팡이츠 파트너 라이더 실적 페이지에 이미 로그인되어 있고 해당 웹페이지가 켜져 있는 PC 환경에서 실시간 실적을 수집하고, Windows 운영 환경에서는 수집 결과를 정해진 카카오톡 단체 채팅방에 텍스트 메시지로 전송한다.

전송 메시지는 사용자가 제공한 예시처럼 카카오톡에서 바로 읽을 수 있는 요약 텍스트 형태를 기준으로 한다. 초보자가 가장 쉽게 구현하고 운영할 수 있도록 이미지 생성 없이 PC 카카오톡 입력창에 텍스트를 붙여넣는 방식을 기본안으로 한다. 작업 스케줄러 없이 UI에서 `시작`을 누르면 즉시 1회 실행하고 이후 기본 35분 간격으로 반복 실행한다.

- 카카오톡 PC 앱 입력창에 텍스트를 붙여넣어 전송해야 하므로 Windows 운영 전송 중에는 사용자 화면 세션이 유지되어야 한다.
- 계정 ID, 비밀번호, OTP, 인증 쿠키를 코드나 문서에 저장하지 않는다.

## 4. 접근안

### 4.1 권장안: 로컬 PC 자동화 방식

Playwright와 Scrapling으로 사이트 데이터를 수집하고, 생성된 요약 텍스트를 Windows용 카카오톡 PC 앱에 클립보드로 붙여넣어 전송한다. 실행 주기와 설정값은 UI에서 관리한다.

장점:
- 사용자가 이미 로그인한 상태를 그대로 활용할 수 있다.
- 카카오톡 공식 API 승인이나 토큰 발급이 없어도 시작할 수 있다.
- 요청한 “로그인된 상태 + UI 시작 버튼 + 35분 반복” 운영 방식과 가장 잘 맞는다.

단점:
- PC 화면 세션과 카카오톡 UI 상태에 영향을 받는다.
- 카카오톡 UI 변경, 창 최소화, 채팅방명 변경에 취약할 수 있다.

본 사양은 이 방식을 기준으로 한다.

### 4.2 대안: 카카오 Developers 메시지 API

공식 API로 메시지를 보내는 방식이다.

장점:
- UI 자동화보다 안정적이다.
- 서버 환경에서도 실행하기 쉽다.

단점:
- 앱 등록, OAuth 토큰, 권한 설정이 필요하다.
- 일반 카카오톡 단체방에 자유롭게 자동 발송하는 용도에는 제약이 있을 수 있다.

### 4.3 대안: 알림 채널 변경

카카오워크, Slack, Telegram 등 웹훅 기반 채널을 사용한다.

장점:
- 자동화 안정성이 높다.
- 서버 운영에 적합하다.

단점:
- 사용자가 요청한 카카오톡 전송과 다르다.

## 5. 실행 흐름

1. 사용자가 UI에서 설정값을 입력하고 `시작`을 누른다.
2. 프로그램이 즉시 1회 실행한다.
3. 실행 시작 시 락 파일을 확인해 이전 작업이 아직 실행 중이면 이번 실행을 건너뛴다.
4. Playwright가 CDP로 원격 디버깅 포트의 Chrome에 연결한다.
5. 열린 탭 중 쿠팡이츠 라이더 실적 페이지를 찾아 로그인 상태를 그대로 활용한다.
6. 페이지 로딩 완료, 네트워크 안정화, 주요 실적 영역 표시를 기다린다.
7. Playwright locator로 주요 DOM을 확보하고, 필요 시 Scrapling Selector로 HTML을 파싱한다.
8. 수집값을 내부 데이터 모델로 정규화한다.
9. 필수 값 누락, 숫자 형식 오류, 시간 불일치를 검증한다.
10. 메시지 템플릿에 데이터를 넣어 전송할 텍스트를 생성한다.
11. 카카오톡 PC 앱에서 대상 단체 채팅방을 열고 텍스트를 붙여넣어 전송한다.
12. 성공/실패 로그와 필요 시 디버그 스크린샷을 저장한다.
13. `중지`를 누르기 전까지 UI 설정의 실행 간격, 기본 35분에 맞춰 반복한다.

## 8. 크롤링 상세

### 8.1 브라우저 세션

- 기본값은 `connect_over_cdp()`로 원격 디버깅 포트의 Chrome에 연결한다.
- macOS 사용자는 UI의 `앱 실행 준비하기(mac)` 버튼으로 전용 Chrome 창을 실행할 수 있다.
- 사용자는 Chrome을 `--remote-debugging-port=9222` 옵션으로 실행하고, 그 창에서 직접 로그인한다.
- 크롤러는 열린 탭 중 실적 URL과 같은 탭을 찾아 `page.content()`를 읽는다.
- 자동 로그인, OTP 처리, 인증 쿠키 저장은 하지 않는다.
- `launch_persistent_context()` 방식은 `browser_mode=persistent` 예비 모드로만 유지한다.

### 8.2 로딩 대기

다음 조건을 모두 만족할 때 수집을 시작한다.

- URL이 대상 페이지와 일치한다.
- 실적 영역 컨테이너가 화면에 표시된다.
- 기준 시간 텍스트가 존재한다.
- 배정/처리/거절율/수행중인인원 중 최소 1개 이상의 핵심 지표가 표시된다.
- 네트워크 idle 또는 별도 타임아웃이 완료된다.

### 8.3 파싱 원칙

- Playwright locator를 1차 수집 수단으로 사용한다.
- HTML 조각을 가져온 뒤 Scrapling Selector로 CSS/XPath 파싱을 보강한다.
- 화면 표시 텍스트 기준 파싱을 우선하되, 가능한 경우 안정적인 `data-*`, aria label, table 구조를 우선한다.
- 숫자는 `건`, `%`, 공백, 쉼표를 제거한 뒤 타입 변환한다.
- 양식 이미지 기준으로 대상 페이지에 존재하지 않는 배정/처리 값은 `https://partner.coupangeats.com/page/peak-dashboard`에서 가져온다.
- 화면 구조 변경에 대비해 파싱 실패 시 전체 페이지 스크린샷과 HTML 스냅샷을 저장한다.

## 9. 카카오톡 텍스트 전송 상세

### 9.1 전송 방식

초보자에게 가장 쉬운 방식은 텍스트 메시지를 Windows 클립보드에 복사한 뒤 카카오톡 PC 앱 대상 단체 채팅방에 붙여넣고 Enter로 전송하는 것이다.

공식 Kakao Developers 메시지 API는 `나에게 보내기` 또는 같은 서비스 내 친구 간 메시지 발송 중심이며, 기존 카카오톡 단체 채팅방에 스케줄러가 자동으로 메시지를 넣는 용도로 쓰기 어렵다. 따라서 이 프로젝트의 기본안은 공식 API가 아니라 Windows PC 카카오톡 UI 자동화다.

### 9.2 UI 설정

- 대상 단체 채팅방명은 UI의 `카카오톡 채팅방명`에 설정한다.
- 실행 간격, 전송 여부, 로그 경로, 브라우저 연결 방식, CDP 주소도 UI에서 설정한다.
- macOS에서는 `앱 실행 준비하기(mac)` 버튼으로 CDP Chrome 실행을 자동화한다.
- 설정값은 `runtime/state/ui_settings.json`에 저장한다.
- Windows 환경변수나 `.env` 파일은 기본 운영 경로로 사용하지 않는다.

### 9.3 대상 채팅방

- 전송 전 카카오톡 검색창에서 채팅방명을 검색해 대상 방을 연다.
- 같은 이름의 채팅방이 여러 개면 오발송 위험이 있으므로 채팅방명을 고유하게 바꾼다. 예: `실적봇_의정부남부`

### 9.4 초보자용 준비사항

1. macOS 또는 Windows PC에 Python 3.10 이상을 설치한다.
2. Chrome을 설치하고, Windows 운영 전송 PC에는 카카오톡 PC 앱도 설치한다.
3. macOS는 `앱 실행 준비하기(mac)` 버튼으로 Chrome을 열고, Windows는 Chrome을 원격 디버깅 포트로 실행한 뒤 쿠팡이츠 파트너 사이트에 직접 로그인해 두고 실적 페이지를 열어둔다.
4. Windows 운영 전송 PC에서는 카카오톡 PC 앱에 직접 로그인해 둔다.
5. 메시지를 받을 단체 채팅방 이름을 고유하게 만든다.
6. 테스트용 단체 채팅방을 먼저 만들고 3회 이상 전송 테스트를 한다.
7. UI에서 `카카오톡 전송`을 끈 상태로 `1회 실행`을 눌러 메시지 미리보기를 확인한다.
8. 정상 메시지가 확인되면 `카카오톡 전송`을 켜고 테스트방에서 전송을 확인한다.
9. PC가 잠금 화면이거나 절전 상태이면 UI 자동화가 실패할 수 있으므로 운영 시간에는 화면 세션이 유지되도록 설정한다.

### 9.5 초보자용 전송 절차

1. 프로그램이 메시지 텍스트를 만든다.
2. `pyperclip.copy(message)`로 텍스트를 클립보드에 복사한다.
3. `pywinauto` 또는 `pyautogui`로 카카오톡 창을 앞으로 가져온다.
4. 카카오톡 검색창에서 `KAKAO_CHAT_NAME`을 검색한다.
5. 검색된 채팅방을 연다.
6. 입력창에 `Ctrl+V`로 메시지를 붙여넣는다.
7. `Enter`로 전송한다.
8. 성공/실패를 로그에 남긴다.

### 9.6 중복 전송 방지

- 실행마다 `captured_at`과 핵심 지표를 해시로 저장한다.
- 직전 전송과 동일한 해시이면 기본값은 전송한다. 운영자가 원하면 `SEND_ONLY_ON_CHANGE=true`로 변경할 수 있다.
- 실패 후 재시도는 1회만 수행한다.

## 11. UI 실행 사양

- 기본 실행 명령:

```bat
cd /d C:\rider_crawl
.venv\Scripts\python.exe -m rider_crawl
```

- `시작`: 즉시 1회 실행 후 UI 설정의 실행 간격마다 반복
- `중지`: 다음 반복 실행을 중단
- `1회 실행`: 반복 없이 즉시 한 번만 실행
- `설정 저장`: 현재 UI 입력값을 JSON에 저장
- 중복 실행: 애플리케이션 락 파일로 방지

UI 없이 저장된 설정으로 1회만 실행할 때:

```bat
cd /d C:\rider_crawl
.venv\Scripts\python.exe -m rider_crawl --once
```

작업 스케줄러는 필수 요건이 아니다.

## 12. 오류 처리

| 상황 | 처리 |
| --- | --- |
| 쿠팡이츠 로그인 만료 | 전송하지 않고 오류 로그, 스크린샷 저장 |
| 페이지 로딩 실패 | 1회 새로고침 후 재시도, 실패 시 종료 |
| 필수 지표 누락 | 전송하지 않고 HTML/스크린샷 저장 |
| 카카오톡 앱 미실행 | 앱 실행 시도 후 1회 재시도 |
| 대상 채팅방 미발견 | 전송 생략, 오류 로그 저장 |
| 메시지 입력창 포커스 실패 | 전송하지 않고 오류 로그 저장 |
| 이전 작업 실행 중 | 이번 실행 건너뜀 |

## 13. 보안과 운영 주의사항

- 계정 비밀번호와 OTP는 저장하지 않는다.
- 원격 디버깅 포트는 로컬 PC에서만 접근 가능하게 두고 외부 네트워크에 노출하지 않는다.
- 예비 `persistent` 모드를 쓰는 경우 브라우저 프로필 디렉터리는 해당 OS 사용자만 접근하도록 권한을 제한한다.
- 수집한 실적 메시지는 내부 운영 목적에만 사용한다.
- 쿠팡이츠 파트너 페이지 접근 권한이 있는 계정에서만 실행한다.
- 사이트 구조 변경 또는 약관 변경 시 크롤링 방식을 재검토한다.
- 실패 스크린샷에는 민감 정보가 포함될 수 있으므로 보관 기간을 제한한다.

## 15. 테스트 기준

### 15.1 단위 테스트

- 저장된 HTML fixture에서 모든 필드를 정확히 파싱한다.
- `10건/19건`, `거절율: 6.2%` 같은 텍스트를 숫자로 변환한다.
- 필수 필드 누락 시 명확한 예외를 발생시킨다.
- 메시지 템플릿이 예시 형식과 동일한 줄 순서로 생성된다.

### 15.2 통합 테스트

- 원격 디버깅 포트로 실행한 실제 Chrome에서 로그인된 대상 탭을 읽는다.
- 텍스트 메시지가 생성되고 사람이 읽을 수 있는지 확인한다.
- UI에서 `카카오톡 전송`을 끈 상태로 카카오톡 전송 없이 전체 플로우가 완료된다.
- 테스트 채팅방에서 실제 텍스트 전송이 성공한다.

### 15.3 운영 검증

- UI `시작` 후 35분 주기 3회 이상 정상 실행된다.
- UI `중지` 후 다음 반복 실행이 발생하지 않는다.
- 이전 실행이 오래 걸릴 때 다음 실행이 중복으로 동작하지 않는다.
- 로그인 만료 상태에서 잘못된 실적 메시지를 보내지 않는다.
- 카카오톡 대상 채팅방을 찾지 못하면 전송하지 않는다.

## 16. 수락 기준

- UI에서 `시작`을 누르면 즉시 실행되고, 기본 35분마다 프로그램이 반복 실행된다.
- UI에서 `중지`를 누르면 다음 반복 실행이 중단된다.
- 로그인된 쿠팡이츠 파트너 페이지에서 실적 데이터를 수집한다.
- 예시와 같은 필드 순서의 텍스트 메시지가 생성된다.
- 생성된 텍스트가 지정된 카카오톡 단체 채팅방에 전송된다.
- 로그인 만료, 파싱 실패, 카카오톡 전송 실패 시 잘못된 메시지를 보내지 않고 로그와 디버그 파일을 남긴다.

- 실패 알림을 별도 채팅방으로 보낼지 여부
