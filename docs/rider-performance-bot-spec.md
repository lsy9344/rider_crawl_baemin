## 1. 목적

배민 달성현황(beta) 또는 쿠팡이츠 실적 페이지에 이미 로그인되어 있고 해당 웹페이지가 켜져 있는 PC 환경에서 실시간 실적을 수집하고, 수집 결과를 정해진 **텔레그램 그룹방** 또는 **카카오톡 단체 채팅방**에 텍스트 메시지로 전송한다.

전송 메시지는 메신저에서 바로 읽을 수 있는 요약 텍스트 형태를 기준으로 한다. 작업 스케줄러 없이 UI에서 `시작`을 누르면 즉시 1회 실행하고 이후 설정한 메세지 전송 간격(분)마다 반복 실행한다.

기본 전송 방식은 **텔레그램 봇 API**다. 텔레그램은 OS에 무관하게 안정적으로 자동 전송할 수 있고, 봇이 받은 그룹방 메시지로 **라이더 조회 명령**과 **키워드 감지 자동응답**까지 처리할 수 있어 권장한다. 카카오톡 전송은 Windows PC 앱 UI 자동화 방식으로 보조 지원한다.

탭은 최대 9개까지 독립적으로 동작하며, 탭마다 플랫폼(배민/쿠팡이츠), 계정 세션, 전송 대상, 실행 간격을 따로 설정한다.

## 2. 전제와 범위

### 2.1 전제

- 크롤링, 메시지 생성, UI 미리보기, 텔레그램 전송은 macOS와 Windows에서 실행할 수 있다.
- 카카오톡 자동 전송은 Windows PC에서만 실행한다(PC 앱 UI 자동화).
- 대상 파트너/센터 사이트 로그인은 사용자가 미리 완료하고, 대상 페이지는 열린 상태로 둔다.
- 쿠팡이츠 파트너 사이트는 로그인 시 2차 인증이 있으므로 자동 로그인은 구현하지 않는다(쿠팡 Gmail 로그인 복구 보조 절차는 별도).
- 텔레그램 전송 시: 봇 토큰과 그룹방 chat_id를 사용자가 발급·확인해 UI에 입력한다. 봇은 대상 그룹방에 초대되어 있어야 하며, 그룹방 메시지를 읽어 명령/키워드를 처리하려면 BotFather에서 **Privacy Mode를 꺼야** 한다.
- 카카오톡 전송 시: Windows용 카카오톡 PC 앱을 사용자가 미리 로그인하고, 메시지를 받을 단체 채팅방이 준비되어 있다. 카카오톡 입력창에 붙여넣어 전송하므로 운영 중에는 화면 세션이 유지되어야 한다.
- 사용자는 UI에서 필요한 설정값을 입력하고 탭별로 `시작`을 누른다.
- 계정 ID, 비밀번호, OTP, 인증 쿠키를 코드나 저장소에 커밋하지 않는다.

### 2.2 포함 범위

- 배민 달성현황(beta) / 쿠팡이츠 실적·피크 대시보드 페이지 접속
- 실적 지표 크롤링 및 정규화(플랫폼별 파서)
- 전송용 텍스트 메시지 생성
- 텔레그램 봇 API로 그룹방(필요 시 토픽) 전송
- 카카오톡 PC 앱 대상 단체 채팅방으로 텍스트 전송(Windows)
- 텔레그램 그룹방 `!이름+번호4자리` 라이더 조회 명령 응답(배민 탭)
- 텔레그램 그룹방 키워드 감지 자동응답(`config.json` 기반)
- 탭(최대 9개)별 UI 설정 저장, 시작/중지 제어, 주기 실행, 중복 실행 방지, 실행 로그 저장

### 2.3 제외 범위

- 파트너/센터 계정 자동 로그인(쿠팡 2차 인증 우회)
- CAPTCHA, OTP, 본인인증 자동 우회
- 카카오톡 계정 자동 로그인
- 임의의 외부 사용자에게 대량 발송
- 대상 사이트의 접근 권한 또는 서비스 약관을 우회하는 행위

## 4. 접근안

### 4.1 권장안(기본 채택): 텔레그램 봇 API 전송

Playwright/crawl4ai로 로그인된 탭 데이터를 수집하고, 생성된 요약 텍스트를 텔레그램 봇 API(`sendMessage`)로 대상 그룹방(필요 시 토픽)에 전송한다. 같은 봇이 `getUpdates`로 그룹방 메시지를 받아 라이더 조회 명령과 키워드 자동응답을 처리한다.

장점:
- OS(Windows/macOS)와 화면 세션 상태에 무관하게 안정적으로 전송된다.
- UI 자동화가 아니므로 창 최소화·포커스·채팅방명 변경에 영향받지 않는다.
- 수신(명령/키워드) 기능을 같은 파이프라인에서 함께 처리할 수 있다.

단점:
- 봇 토큰·그룹방 chat_id 발급이 필요하다.
- 그룹방 메시지 수신 기능은 BotFather Privacy Mode를 꺼야 한다.

**본 사양은 이 방식을 기준으로 한다.**

### 4.2 보조안: 카카오톡 PC 앱 UI 자동화(Windows)

생성된 요약 텍스트를 Windows 클립보드에 복사한 뒤 카카오톡 PC 앱 대상 채팅방에 붙여넣고 Enter로 전송한다. 사용자가 이미 로그인한 카카오톡 환경을 그대로 활용하며 토큰 발급이 필요 없다. 단, PC 화면 세션과 카카오톡 UI 상태에 의존하고 라이더 조회/키워드 자동응답은 지원하지 않는다(텔레그램 탭 전용).

### 4.3 대안: 카카오 Developers 메시지 API

공식 API는 `나에게 보내기`/친구 간 발송 중심이라 기존 단체방 자동 발송에는 제약이 있어 채택하지 않는다.

## 5. 실행 흐름

1. 사용자가 UI 탭에서 설정값(플랫폼·전송 방식 포함)을 입력하고 해당 탭의 `시작`을 누른다.
2. 프로그램이 즉시 1회 실행한다. 전송 방식이 텔레그램이고 토큰·chat_id가 있으면 같은 봇 토큰으로 수신 폴러(getUpdates)도 함께 띄운다.
3. 실행 시작 시 락 파일을 확인해 이전 작업이 아직 실행 중이면 이번 실행을 건너뛴다.
4. Playwright/crawl4ai가 CDP로 원격 디버깅 포트의 Chrome에 연결한다.
5. 열린 탭 중 대상 실적/달성현황 페이지를 찾아 로그인 상태를 그대로 활용한다(쿠팡은 센터 전환·피크 대시보드도 함께 확인).
6. 페이지 로딩 완료, 네트워크 안정화, 주요 실적 영역 표시를 기다린다.
7. Playwright locator로 주요 DOM을 확보하고, 필요 시 HTML을 추가 파싱한다.
8. 수집값을 플랫폼별 데이터 모델로 정규화한다.
9. 필수 값 누락, 숫자 형식 오류, 시간 불일치를 검증한다.
10. 메시지 템플릿에 데이터를 넣어 전송할 텍스트를 생성한다.
11. 전송 방식에 따라 텔레그램 봇 API(`sendMessage`)로 대상 그룹방/토픽에 보내거나, 카카오톡 PC 앱 대상 채팅방에 붙여넣어 전송한다.
12. 성공/실패 로그와 필요 시 디버그 스크린샷을 저장한다.
13. `중지`를 누르기 전까지 UI 설정의 메세지 전송 간격(분)에 맞춰 반복한다.
14. (텔레그램 탭) 수신 폴러는 그룹방 메시지를 받아 `!이름+번호4자리` 라이더 조회 명령을 먼저 처리하고, 명령이 아니면 키워드 감지 자동응답을 검사한다.

## 7. 텍스트 메시지 템플릿

메시지 본문은 전송 방식(텔레그램/카카오톡)과 무관하게 동일한 요약 텍스트를 사용한다. 아래는 쿠팡이츠 기준 예시이며, 배민 탭은 달성현황(beta) 항목에 맞는 별도 템플릿을 사용한다.

### 7.1 기본 형식

```text
[실시간 실적봇]
⏰ {HH:mm} 기준

아침 : {morning_status}
점심 피크 : {lunch_peak_status}
점심 논피크 : {lunch_non_peak_done}건/{lunch_non_peak_total}건
저녁 피크 : {dinner_peak_done}건/{dinner_peak_total}건
저녁 논피크 : {dinner_non_peak_done}건/{dinner_non_peak_total}건

배정 {assigned_count}건 / 처리 {processed_count}건
🚨거절율: {reject_rate}%🚨
🐱수행중인인원 : {active_riders}명
```

### 7.2 현재 화면 기준 예시

```text
[실시간 실적봇]
⏰ 14:02 기준

오후논피크 : 7명/25명
대기 : 0명

완료 : 102.4건
거절/무시 : 2.4건
취소 : 0건
점심피크 : 60.6건
저녁피크 : 0건
논피크 : 41.8건
수행중인인원 : 5명
```

이미지 전송은 초보자용 기본 범위에서 제외한다. 텍스트 전송이 안정화된 뒤 필요하면 `Pillow`로 PNG를 생성하고 클립보드 이미지 전송을 추가한다.

## 8. 크롤링 상세

### 8.1 브라우저 세션

- 기본값은 `connect_over_cdp()`로 원격 디버깅 포트의 Chrome에 연결한다.
- macOS 사용자는 UI의 `앱 실행 준비하기(mac)` 버튼으로 전용 Chrome 창을 실행할 수 있다.
- 사용자는 Chrome을 `--remote-debugging-port=9222` 옵션으로 실행하고, 그 창에서 직접 로그인한다.
- 크롤러는 열린 탭 중 실적 URL과 같은 탭을 찾아 `page.content()`를 읽는다.
- 자동 로그인, OTP 처리, 인증 쿠키 저장은 하지 않는다.
- `launch_persistent_context()` 방식은 `browser_mode=persistent` 예비 모드로만 유지한다.

### 8.2 로딩 대기

다음 조건을 모두 만족할 때 수집을 시작한다.

- URL이 대상 페이지와 일치한다.
- 실적 영역 컨테이너가 화면에 표시된다.
- 기준 시간 텍스트가 존재한다.
- 배정/처리/거절율/수행중인인원 중 최소 1개 이상의 핵심 지표가 표시된다.
- 네트워크 idle 또는 별도 타임아웃이 완료된다.

### 8.3 파싱 원칙

- Playwright locator를 1차 수집 수단으로 사용한다.
- HTML 조각을 가져온 뒤 crawl4ai로 CSS/XPath 파싱을 보강한다.
- 화면 표시 텍스트 기준 파싱을 우선하되, 가능한 경우 안정적인 `data-*`, aria label, table 구조를 우선한다.
- 숫자는 `건`, `%`, 공백, 쉼표를 제거한 뒤 타입 변환한다.
- 쿠팡이츠 탭에서 실적 페이지에 존재하지 않는 배정/처리 값은 피크 대시보드(`https://partner.coupangeats.com/page/peak-dashboard`)에서 가져온다. 배민 탭은 달성현황(beta) 페이지 한 곳에서 수집한다.
- 화면 구조 변경에 대비해 파싱 실패 시 전체 페이지 스크린샷과 HTML 스냅샷을 저장한다.

## 9. 텍스트 전송 상세

탭마다 `전송 방식`을 텔레그램 또는 카카오톡 중 하나로 선택한다. 텔레그램이 기본·권장이며, 카카오톡은 Windows 보조 방식이다.

### 9.1 텔레그램 전송(기본)

- 봇 토큰과 그룹방 `chat_id`로 텔레그램 봇 API `sendMessage`를 호출해 전송한다. 토픽 그룹이면 `telegram_message_thread_id`로 토픽까지 지정한다.
- 같은 봇 토큰을 여러 탭이 공유하면 수신 폴러(getUpdates)는 토큰당 하나만 띄우고, 그 토큰을 쓰는 활성 탭 전부를 명령 라우팅 대상으로 등록한다. 폴러가 둘로 나뉘어 명령이 누락되는 것을 막는다.
- 준비:
  1. BotFather로 봇을 만들고 토큰을 발급한다.
  2. 봇을 대상 그룹방에 초대한다.
  3. 라이더 조회 명령·키워드 자동응답처럼 그룹방 메시지를 받으려면 BotFather `/setprivacy`에서 **Privacy Mode를 Disable** 한다.
  4. 그룹방에 아무 메시지나 한 번 보낸 뒤 `getUpdates`로 `chat_id`(토픽이면 `message_thread_id`도)를 확인해 UI에 입력한다.
- 전송 방식이 텔레그램이고 토큰·chat_id가 있으며 `메시지 전송`이 켜진 탭에서만 전송·수신 파이프라인이 동작한다.

### 9.2 카카오톡 전송(Windows 보조)

- 텍스트 메시지를 Windows 클립보드에 복사한 뒤 카카오톡 PC 앱 대상 단체 채팅방에 붙여넣고 Enter로 전송한다.
- 대상 단체 채팅방명은 UI의 `카카오톡 채팅방명`에 설정한다. 같은 이름의 방이 여러 개면 오발송 위험이 있으므로 채팅방명을 고유하게 바꾼다. 예: `실적봇_의정부남부`
- 카카오톡 전송은 라이더 조회 명령·키워드 자동응답을 지원하지 않는다(텔레그램 탭 전용). macOS에서는 카카오톡 전송이 자동으로 비활성화된다.
- 절차: 메시지 생성 → 클립보드 복사 → 카카오톡 창 활성화 → 채팅방명 검색 → 방 열기 → `Ctrl+V` → `Enter` → 성공/실패 로그.

### 9.3 공통 UI/설정

- 전송 방식, 메세지 전송 간격, 전송 여부, 변경 시에만 전송 여부, 로그 경로, CDP 주소는 UI에서 탭별로 설정한다.
- macOS에서는 `앱 실행 준비하기(mac)` 버튼으로 CDP Chrome 실행을 자동화한다.
- 설정값은 `runtime/state/ui_settings.json`에 저장한다(저장소에 커밋하지 않는 로컬 상태 파일).
- 키워드 자동응답 문구·키워드·쿨다운은 프로그램 옆 `config.json`에서 읽는다(§9.5).

### 9.4 중복 전송 방지

- 실행마다 `captured_at`과 핵심 지표를 해시로 저장한다.
- 직전 전송과 동일한 해시이면 기본값은 전송한다. 운영자가 원하면 `변경 시에만 전송`을 켜서 동일 해시는 건너뛸 수 있다.
- 실패 후 재시도는 1회만 수행한다.

### 9.5 라이더 조회 명령(텔레그램, 배민 탭)

- 그룹방에서 `!이름+휴대폰뒤4자리`(예: `!홍길동1234`) 형식을 보내면 봇이 배민 배달현황 테이블을 다시 크롤링해 해당 라이더의 취소 데이터를 답장한다.
- 명령은 들어온 채팅방/토픽에 연결된 탭만 조회한다. 다른 탭의 라이더는 함께 조회하지 않는다.
- 취소율 = `총 취소 수 / (완료 + 거절 + 총 취소 수) * 100`, 총 취소 수 = `배차취소 + 배달취소(라이더귀책)`. 4% 이상이면 `위험합니다.`, 아니면 `정상 범위입니다.`
- 라이더를 찾지 못하면 `해당 라이더를 찾지 못했습니다.`라고 답장한다.
- 라이더 조회는 배민 탭에서만 지원한다. 쿠팡이츠 탭 채팅방에서 보내면 `라이더 조회 명령은 배민 탭에서만 지원합니다.`라고 답장한다.

### 9.6 키워드 감지 자동응답(텔레그램)

- 그룹방 메시지에 특정 키워드(기본값 `사고`, `병원`)가 **부분 일치**로 포함되면, `config.json`의 안내 메시지를 같은 채팅방(토픽이 있으면 같은 토픽)으로 발송한다.
- 별도 프로세스가 필요 없다. 텔레그램 봇 토큰·chat_id가 입력되고 전송이 켜진 텔레그램 탭을 `시작`하면 수신 파이프라인에 함께 동작한다(카카오톡 탭은 대상이 아니다).
- UI에 입력한 `채팅방 ID`(+토픽 ID)와 일치하는 방/토픽에서만 감지·응답한다.
- `/`로 시작하는 메시지는 제외하고, `!이름+번호` 라이더 조회 명령은 키워드보다 먼저 처리한다.
- 같은 대상에서 마지막 **전송 성공** 후 `cooldown_seconds`(기본 30초) 이내 반복 키워드는 응답하지 않는다. 전송 실패 시 쿨다운을 기록하지 않아 다음 메시지에서 다시 시도한다.
- `config.json` 위치: 개발/CLI 실행은 작업 디렉터리(프로젝트 루트), exe(onefile)는 exe 옆 폴더. 메시지마다 다시 읽으므로 **재시작 없이** 수정이 반영된다.

```json
{
  "keywords": ["사고", "병원"],
  "auto_message": "☎️KB생명 :1111-2222\n☎️삼성생명 : 2222-3333\n☎️한화생명 : 3333-4444\n☎️DB보험 : 8888-9999",
  "cooldown_seconds": 30
}
```

## 10. UI 설정값

UI는 탭(최대 9개)으로 구성되며, 각 탭이 아래 항목을 독립적으로 갖는다.

- 플랫폼: `배민`(baemin) / `쿠팡이츠`(coupang). 기본 `배민`
- 전송 방식(메신저): `텔레그램`(telegram) / `카카오톡`(kakao). 기본 `텔레그램`
- 실적/달성현황 URL: 플랫폼에 맞는 대상 페이지 URL
- 보조 URL(쿠팡 피크 대시보드): 쿠팡이츠 탭에서 필수, 배민 탭은 미사용
- 센터명: 배민 센터명 / 쿠팡 기대 센터·상점명
- 배민 센터 ID: 배민 탭에서 사용(쿠팡 미사용)
- CDP 주소: 탭별 기본 `http://127.0.0.1:9222`, `9223`, … 으로 분리
- 앱 전용 브라우저 프로필 경로: 탭별 `runtime/browser-profile`, `-2`, … 으로 분리
- 텔레그램 봇 토큰 / 채팅방 ID / 토픽 ID(선택): 텔레그램 전송 시 입력
- 카카오톡 채팅방명: 카카오톡 전송 시 입력
- 로그 경로: 기본 `logs`
- 메세지 전송 간격: 분 단위
- 페이지 타임아웃: 기본 `60000`ms
- 락 타임아웃: 기본 `60`초
- Headless 여부: 기본 꺼짐
- 메시지 전송 여부: 기본 꺼짐(운영 전 켜기)
- 변경 시에만 전송 여부: 기본 꺼짐

> 텔레그램 탭은 `전송 방식=텔레그램`, `메시지 전송=ON`, 봇 토큰·채팅방 ID 입력, 실적 URL 입력 후 `시작`해야 전송·수신(라이더 조회/키워드 자동응답)이 동작한다. `1회 실행`만으로는 수신 폴러가 뜨지 않는다.

설정 저장 위치:

```text
runtime/state/ui_settings.json
```

> `runtime/`은 저장소에 커밋하지 않는 로컬 상태 디렉터리다. 봇 토큰 등 민감값은 이 파일에만 두고 코드/문서에는 저장하지 않는다.

## 11. UI 실행 사양

- 기본 실행 명령:

```bat
cd /d C:\rider_crawl
.venv\Scripts\python.exe -m rider_crawl
```

- `시작`: 즉시 1회 실행 후 UI 설정의 실행 간격마다 반복
- `중지`: 다음 반복 실행을 중단
- `1회 실행`: 반복 없이 즉시 한 번만 실행
- `설정 저장`: 현재 UI 입력값을 JSON에 저장
- 중복 실행: 애플리케이션 락 파일로 방지

UI 없이 저장된 설정으로 1회만 실행할 때:

```bat
cd /d C:\rider_crawl
.venv\Scripts\python.exe -m rider_crawl --once
```

작업 스케줄러는 필수 요건이 아니다.

## 12. 오류 처리

| 상황 | 처리 |
| --- | --- |
| 대상 사이트 로그인 만료 | 전송하지 않고 오류 로그, 스크린샷 저장 |
| 페이지 로딩 실패 | 1회 새로고침 후 재시도, 실패 시 종료 |
| 필수 지표 누락 | 전송하지 않고 HTML/스크린샷 저장 |
| 텔레그램 전송 실패(API 오류/네트워크) | 전송 실패 로그 저장, 재시도 1회 |
| 카카오톡 앱 미실행 | 앱 실행 시도 후 1회 재시도 |
| 카카오톡 대상 채팅방 미발견 | 전송 생략, 오류 로그 저장 |
| 메시지 입력창 포커스 실패(카카오톡) | 전송하지 않고 오류 로그 저장 |
| 이전 작업 실행 중 | 이번 실행 건너뜀 |

## 13. 보안과 운영 주의사항

- 계정 비밀번호, OTP, 텔레그램 봇 토큰은 코드/문서에 저장하지 않는다. 봇 토큰·chat_id는 로컬 `runtime/state/ui_settings.json`에만 둔다.
- 원격 디버깅 포트는 로컬 PC에서만 접근 가능하게 두고 외부 네트워크에 노출하지 않는다.
- 예비 `persistent` 모드를 쓰는 경우 브라우저 프로필 디렉터리는 해당 OS 사용자만 접근하도록 권한을 제한한다.
- 텔레그램 봇은 대상 그룹방에만 초대하고, 그룹 외부에 토큰이 노출되지 않게 한다. Privacy Mode를 끈 봇은 그룹 메시지를 받으므로 대상 그룹 관리에 주의한다.
- 수집한 실적 메시지는 내부 운영 목적에만 사용한다.
- 대상 파트너/센터 페이지 접근 권한이 있는 계정에서만 실행한다.
- 사이트 구조 변경 또는 약관 변경 시 크롤링 방식을 재검토한다.
- 실패 스크린샷에는 민감 정보가 포함될 수 있으므로 보관 기간을 제한한다.

## 15. 테스트 기준

### 15.1 단위 테스트

- 저장된 HTML fixture에서 플랫폼별 필드를 정확히 파싱한다.
- `10건/19건`, `거절율: 6.2%` 같은 텍스트를 숫자로 변환한다.
- 필수 필드 누락 시 명확한 예외를 발생시킨다.
- 메시지 템플릿이 예시 형식과 동일한 줄 순서로 생성된다.
- 키워드 자동응답: 부분 일치 감지, `/`로 시작하는 메시지 제외, 쿨다운(전송 성공 후 기록) 동작을 검증한다.
- 라이더 조회: 명령 파싱, 취소율 계산, 미발견/비배민 탭 응답 문구를 검증한다.

### 15.2 통합 테스트

- 원격 디버깅 포트로 실행한 실제 Chrome에서 로그인된 대상 탭을 읽는다.
- 텍스트 메시지가 생성되고 사람이 읽을 수 있는지 확인한다.
- UI에서 `메시지 전송`을 끈 상태로 전송 없이 전체 플로우가 완료된다.
- 텔레그램 테스트 그룹방에서 실제 텍스트 전송과 `!조회` 명령·키워드 자동응답이 동작한다.

### 15.3 운영 검증

- UI `시작` 후 설정한 메세지 전송 간격으로 3회 이상 정상 실행된다.
- UI `중지` 후 다음 반복 실행이 발생하지 않는다.
- 이전 실행이 오래 걸릴 때 다음 실행이 중복으로 동작하지 않는다.
- 로그인 만료 상태에서 잘못된 실적 메시지를 보내지 않는다.
- 카카오톡 대상 채팅방을 찾지 못하면 전송하지 않는다.
- 텔레그램 탭: `전송 방식=텔레그램`, `메시지 전송=ON`, 토큰·chat_id 입력 후 `시작`하면 수신 폴러가 떠 명령·키워드가 처리된다.

## 16. 수락 기준

- UI 탭에서 `시작`을 누르면 즉시 실행되고, 설정한 메세지 전송 간격마다 반복 실행된다.
- UI에서 `중지`를 누르면 다음 반복 실행이 중단된다.
- 로그인된 배민/쿠팡이츠 대상 페이지에서 실적 데이터를 수집한다.
- 예시와 같은 필드 순서의 텍스트 메시지가 생성된다.
- 생성된 텍스트가 지정된 텔레그램 그룹방 또는 카카오톡 단체 채팅방에 전송된다.
- 텔레그램 탭에서 `!이름+번호4자리` 라이더 조회와 키워드 감지 자동응답이 동작한다.
- 로그인 만료, 파싱 실패, 전송 실패 시 잘못된 메시지를 보내지 않고 로그와 디버그 파일을 남긴다.

## 17. 구현 시 남은 확인사항

- 실제 페이지의 DOM 구조와 안정적인 selector(배민/쿠팡 각각)
- 대상 텔레그램 그룹방 chat_id / 토픽 ID, 또는 카카오톡 채팅방명
- 메시지를 매 실행마다 보낼지, 지표 변경 시에만 보낼지
- 운영 시간 제한이 필요한지 여부
- 실패 알림을 별도 채팅방으로 보낼지 여부
